# 重要な指示（これを最初に配置）

**出力形式の絶対ルール：**
- 出力は「日報Markdownコンテンツ」だけ
- 一行目から指定されたセクション（## 1. 本日の業務概要）から開始
- 日報以外の文字・説明・メッセージは一切含めない

**禁止事項（違反時は失敗と判定）：**
- ✗ 「読み込まれました」「以下の情報から読み取れます」などの説明文
- ✗ 「ご指摘ありがとうございます」などの確認・返答
- ✗ 「参考：」「注：」などの補足説明
- ✗ 「いかがでしょうか」などの質問返し
- ✗ 改行による間隔の挿入（コンテンツ前後の空白）

**以下のルールは絶対に守ってください**

1. 言語：日本語のみ。英語は使用禁止
2. 形式：Markdownのみ
3. 禁止事項：
   - 説明文
   - 確認メッセージ
   - 挨拶・結びの言葉
   - 前置き・後置き

# 依頼概要
以下の情報及び作業ログの添付ファイルを元に、今日1日の活動報告（日報）をMarkdown形式で日本語で作成してください。
回答は、今日1日の活動報告（日報）のみで記載してください。
この回答結果を日報としてそのままコピペするので、余計な前後の応答は記載しないでください。
出力はMarkdownブロックのみ、またはテキストのみとし、挨拶、解説、確認、結びの言葉は一切禁止します。
(｢## 1. 本日の業務概要｣から回答が始まり、｢## 4. 作業詳細｣までを記載したら余計な文言は一切記載しないでください。)
依頼内容に沿ったフォーマットを必ず遵守して記載してください

# 依頼内容
- 今日の作業ログとカレンダーの予定を照らし合わせ、実際に何に時間を使ったかを推測して要約してください。
- プロジェクトごと、または作業カテゴリごとに時間を集計してください。
- 予定されていた会議等の時間に、別の作業（SlackやVS Codeなど）をしていた場合は、その旨を指摘してください。
- ただし、会議に関係する作業の場合は、アプリケーションを起動して作業をしていても指摘は不要です。
- タイトルは以下にして、それぞれの内容を記載してください。
    ## 1. 本日の業務概要
    ## 2. 作業カテゴリ別時間集計
    ## 3. 予定と実績の乖離・指摘事項
    ## 4. 作業詳細
- 上記2, 4はMarkdown表にして、粒度は細かすぎないように記載してください
- 上記4は主要な業務ブロックのみ（1時間以上の作業単位）を時系列リストで記載してください。細かい操作画面の遷移は含めないでください。
